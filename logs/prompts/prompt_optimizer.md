# 🎯 프롬프트 최적화 시스템

## 최적화 원칙

### 1. 모호함 제거
- 구체적 요구사항으로 변환
- 실행 범위 명확화
- 예상 결과 정의

### 2. SuperClaude 활용
- 적절한 MCP 서버 선택
- 페르소나 활성화
- 플래그 활용

### 3. 실행 가능성 검증
- 기술적 실현 가능성
- 리소스 요구사항
- 시간 추정

## 최적화 템플릿

```markdown
### 원본 프롬프트
[사용자 입력]

### 최적화된 프롬프트
**목표**: [구체적 목표]
**범위**: [작업 범위]
**요구사항**:
- [요구사항 1]
- [요구사항 2]

**사용할 도구/프레임워크**:
- SuperClaude: [플래그/페르소나]
- MCP: [서버 목록]
- Tools: [필요 도구]

**예상 결과**: [기대하는 결과]
**확인 필요사항**: [사용자 확인 요청]
```

## 이력 저장 형식

파일명: `YYYYMMDD_HHMMSS_prompt_optimization.md`
